# Code of Conduct

## Our Pledge
We want the Opix SDK community to be a welcoming, safe, and respectful place for everyone.  
By participating in this project, contributors agree to treat others with kindness, patience, and professionalism.

## Our Standards
Examples of behaviour that helps create a positive environment:

- Being respectful and considerate  
- Giving helpful, constructive feedback  
- Asking questions instead of assuming  
- Accepting responsibility for mistakes  
- Focusing on what’s best for the project  

Examples of behaviour that is not acceptable:

- Harassment, insults, or personal attacks  
- Discrimination or disrespect toward any group  
- Sharing harmful, unsafe, or inappropriate content  
- Intentionally derailing discussions  
- Any behaviour that makes others feel unsafe or unwelcome  

## Our Responsibilities
Project maintainers are responsible for:

- Clarifying standards of behaviour  
- Reviewing contributions fairly  
- Taking action when unacceptable behaviour occurs  
- Removing or rejecting contributions that violate this Code of Conduct  

## Scope
This Code of Conduct applies to:

- The GitHub repository  
- Issue discussions  
- Pull requests  
- Community spaces related to the project  

## Enforcement
If you see behaviour that violates this Code of Conduct, you can report it privately to the project maintainer.

Maintainer: **@ThatBobo**

Reports will be handled respectfully and confidentially.

## Consequences
If someone violates this Code of Conduct, maintainers may take appropriate action, including:

- A warning  
- Temporary or permanent removal from the project  
- Reverting or blocking contributions  

## Acknowledgement
This Code of Conduct is inspired by the Contributor Covenant, a widely used standard for open-source communities.
